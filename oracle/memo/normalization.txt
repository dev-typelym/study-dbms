DML(Data Manipulation Language)
	SELECT, UPDATE, INSERT, DELETE

TCL(Transaction Control Language)
	트랜잭션을 제어하는 명령
	COMMIT, ROLLBACK, SAVEPOINT

COMMIT
	하나의 트랜잭션을 정상적으로 완료하고, 그 결과를 
	데이터베이스에 반영하라

ROLLBACK
	문제가 발생한 트랜잭션을 취소하기 위한 명령어

SAVEPOINT
	하나의 트랜잭션을 작게 분할하여 저장하는 명령어

transaction(트랜잭션)
	더이상 쪼갤 수 없는 최소한의 업무 단위
	일종의 논리적인 업무 단위
	가정, A가 B에게 100만원 송금
	A계좌에서 100만원이 들어있는지 조회(SELECT)
	A계좌에서 100만원 차감(UPDATE)
	B계좌에서 100만원 추가(UPDATE)

트랜잭션의 4가지 특징(ACID)
	Atomicity 원자성
		더이상 분해가 불가능한 업무의 최소 단위
		All or Nothing
	Consistency 일관성
		트랜잭션이 실행되기 전과 후의 데이터베이스 내용에 모순이
		있으면 안된다
	Isolation 격리성
		실행중인 트랜잭션 중간 결과에 다른 트랜잭션 접근 불가
	Durability 영속성
		트랜잭션이 성공적으로 완료되었으면, 결과는 영구적으로 
		반영되어야 한다

Transaction
	SELECT
	UPDATE	-->	commit --> 데이터베이스에 영구적으로 반영
	DELETE	--> 	rollback --> 실패(데이터베이스 반영X)
	INSERT

Transcation 실행방식
	AUTO COMMIT 
		기본모드, DML 한 줄을 수행할 때마다
		DBMS가 알아서 트랜잭션을 컨트롤 하는 방식
		(한 줄을 트랜잭션 전부라고 판단하고
		문법적 오류가 없으면 COMMIT, 문법적 오류가 있으면 
		ROLLBACK)
	암시적 트랜잭션
		트랜잭션의 시작은 DBMS가 처리하고
		트랜잭션의 끝은 사용자가 명시적으로 처리하는 방법
	명시적 트랜잭션
		트랜잭션의 시작과 끝을 사용자가 지정하는 방식
		BEGIN TRANSACTION; 으로 시작
		COMMIT/ROLLBACK; 으로 끝

DML을 사용한 후 COMMIT / ROLLBACK은 필수
따라서 조건이나 서브쿼리의 결과를 예상할 수 없다면 
커밋하기 이전에 조회를 하여 원하는 결과가 나왔는지 확인한 후
COMMIT 또는 ROLLBACK하는 것이 바람직하다

DDL(Data Definition Language)
	데이터베이스의 데이터 구조 구성요소 등을 다루는 명령어
	추가 삭제 수정
	
DDL 적용 대상
	테이블(Table)
		데이터를 저장하는 기본 저장 단위
	뷰(View)
		하나 이상의 테이블로 부터 유도되는 가상 테이블
	인덱스(Index)
		검색 속도를 향상시키기 위한 데이터베이스의 장치
	스키마(Schema)
		데이터베이스의 구조와 제약조건에 대한 전반적인
		설명을 기술한 메타데이터들의 집합
	


CREATE
	추가(생성)

	CREATE TABLE 테이블이름(
		컬럼명 타입(크기),
		컬럼명 타입(크기)
	);



데이터 타입
	NUMBER : 숫자
	CHAR : 문자(크기 고정)
		최대크기 2000바이트
		저장된 값이 설정된 크기보다 작다면 나머지 공간이 채워짐
	VARCHAR2 : 문자(크기 가변적)
		최대사이지는 4000바이트
		가변길이로 조정이 되어 실질적으로 사용한 만큼만 크기를 
		차지한다
	DATE : 날짜
		1초단위

그외
	10가지 이상의 숫자 타입을 갖고 있음, NUMERIC 타입이라고 한다
	CHARACTER
		최대 8000바이트
	VARCHAR
		최대 8000바이트
	DATETIME
		3.33밀리초단위

제약조건설정
	컬럼수준에서 제약조건 생성
	CREATE TABLE 테이블명(
		컬럼명 데이터타입(크기) 제약조건 제약조건 제약조건,
		컬럼명 데이터타입(크기) 제약조건
	);

	테이블수준에서 제약조건 생성
	CREATE TABLE 테이블명(
		컬럼명 데이터타입(크기),
		컬럼명 데이터타입(크기),
		CONSTRAINT 제약조건명 제약조건종류 (컬럼명),
		CONSTRAINT 제약조건명 제약조건종류 (컬럼명)
	);
		
제약조건 종류
	PRIMARY KEY
	FORIEGN KEY
	UNIQUE
	NOT NULL
	CHECK

데이터무결성
	데이터베이스에 저장된 값들이 정확하고 일관성있는 데이터임을 나타내는
	특성
	개체무결성(entity integrity)
		각각의 행은 구분이 되어야 한다
		primary key로 선택된 컬럼은 중복되지 않고
		null값을 가질 수 없다
	참조무결성(reference integrity)
		부모에 존재하는 값만 자식에 추가할수 있고
		자식에 사용하지 않는 값만 부모에서 삭제할 수 있다
		foreign key
	영역무결성(domain integrity)
		데이터의 타입, 범위, 기본값, 유일성과 관련된 제한
		정의된 domain에 알맞은 데이터만 컬럼에 저장할 수 있다
		check/데이터타입
	비즈니스무결성(business integrity)
		사용자의 업무 규칙에 따른 비즈니스적인 제약조건
		check
